Metodologia

Dal dataset grezzo del Comune di Bologna al Knowledge Graph arricchito: i passi del progetto.

Flusso di lavoro

  1. Acquisizione dati
    Download degli Open Data del Comune di Bologna: il dataset degli archi stradali (rifter_arcstra, 1.960 record) e il dataset "Vie dedicate alle donne" (157 record). I dati sono in formato CSV.
  2. Classificazione di genere
    Per ogni strada è stato identificato il referente (persona, luogo, concetto) e assegnata la categoria Male, Female o Toponimo. Questa operazione — la più impegnativa — è stata condotta con l'ausilio di LLM (DeepSeek, Gemini, ChatGPT, Claude, Copilot) e validata manualmente.
  3. Costruzione del Knowledge Graph
    Lo script genera_ttl.py (Python + rdflib) legge il CSV classificato e genera il file bologna_KG_corretto.ttl in formato Turtle (6.783 triple). Ogni strada è un nodo clv:Street; le strade dedicate a persone sono collegate tramite clv:isDedicatedTo a un nodo cpv:Person.
  4. Arricchimento biografico via Wikidata
    Per le ~470 persone identificate, lo script wikidata_fetch.py interroga l'API JSON di Wikidata recuperando professione (P106), data/luogo di nascita (P569/P19) e data/luogo di morte (P570/P20). I dati sono salvati nel file F_M DATE.xlsx.
  5. Classificazione professionale
    Lo script classifica_professioni.py assegna una delle 12 macro-categorie a ciascuna delle 1.192 persone classificate (1.091 uomini + 67 donne storiche + 34 proposte), usando il matching per parole chiave sulle professioni Wikidata.
  6. Proposte di intitolazione
    Il gruppo di ricerca ha identificato e documentato 34 candidature (29 attive + 5 future) per nuove intitolazioni a donne, persone trans e non binarie con legame con Bologna o rilevanza nazionale. I dati sono in proposte_intitolazioni_future.csv e proposte_KG.ttl.

Struttura del Knowledge Graph

Il KG utilizza due ontologie del Semantic Web italiano:

  • CLV (Core Location Vocabulary) — per modellare le strade
  • CPV (Core Person Vocabulary) — per modellare le persone

Esempio di triple per una strada femminile:

<https://w3id.org/bologna/resource/street/abc123>
    a clv:Street ;
    clv:hasStreetName "VIA LAURA BASSI" ;
    clv:isDedicatedTo <https://w3id.org/bologna/resource/person/def456> .

<https://w3id.org/bologna/resource/person/def456>
    a cpv:Person ;
    cpv:fullName "Laura Bassi Veratti" ;
    cpv:sex "Female" .

Per le proposte di intitolazione, abbiamo esteso il vocabolario con una proprietà custom:

@prefix ex: <https://w3id.org/bologna/ontology#> .

<https://w3id.org/bologna/resource/person/...>
    a cpv:Person ;
    cpv:fullName "Maria Dalle Donne" ;
    cpv:sex "Female" ;
    ex:proposta "true" ;
    ex:statoProposta "attiva" .

Visualizzazione dell'ontologia (WebVOWL)

Lo schema delle classi e proprietà è disponibile nel file schema.ttl in formato OWL/Turtle. Puoi esplorarlo interattivamente con WebVOWL, uno strumento open source per la visualizzazione grafica di ontologie OWL sviluppato dall'Università di Stoccarda.

Apri lo schema in WebVOWL →

WebVOWL carica schema.ttl da GitHub Pages e mostra le classi (Street, Person), la proprietà oggetto isDedicatedTo e le proprietà dato (hasStreetName, sex, fullName, proposta, statoProposta) come grafo interattivo navigabile.

Strumenti utilizzati

Python 3 + rdflib

Generazione e manipolazione del KG in formato Turtle. rdflib è la libreria Python standard per il Semantic Web.

openpyxl

Lettura e scrittura del file F_M DATE.xlsx con i dati biografici delle 1.160 persone censite.

Wikidata API

Recupero automatico di professioni e date biografiche per ~470 persone tramite wbsearchentities e wbgetentities.

SPARQL-Anything

Tool Java per eseguire query SPARQL CONSTRUCT su file CSV, usato nelle fasi iniziali del progetto per generare le triple RDF.

LLM (5 modelli)

DeepSeek, Gemini, ChatGPT, Claude e Copilot per la classificazione di genere e l'arricchimento del KG. Vedi la sezione LLM & Prompt per il confronto.

Chart.js + HTML/CSS/JS

Visualizzazioni interattive in questo sito. Tutto il codice è pubblicamente disponibile nel repository GitHub.

Note metodologiche complete
