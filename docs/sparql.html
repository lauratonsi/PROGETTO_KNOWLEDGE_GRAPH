Query SPARQL sul Knowledge Graph

Cinque query che interrogano il KG bolognese. Utilizzano tutte le keyword obbligatorie: OPTIONAL, DISTINCT, UNION, FILTER, REGEX, LIMIT, ORDER BY.

Il Knowledge Graph è disponibile come file Turtle nel repository: bologna_KG_corretto.ttl (6.783 triple). Per eseguire le query localmente: python run_queries.py (usa rdflib).

Query 1 — Conteggio strade per genere

Domanda: quante strade di Bologna sono dedicate a uomini e quante a donne? Questa è la risposta principale al gender gap.
Keyword: DISTINCT, COUNT, GROUP BY, ORDER BY

"su">"sc">## Query 1 — Conteggio strade per genere"su">
"su">"sc">## Keywords: DISTINCT, COUNT, GROUP BY, ORDER BY"su">
"su">"sc">##"su">
"su">"sc">## Domanda: quante strade di Bologna sono dedicate a uomini"su">
"su">"sc">## e quante a donne? Questa è la risposta principale al gender gap."su">

PREFIX clv: <https://w3id.org/italia/onto/CLV/>
PREFIX cpv: <https://w3id.org/italia/onto/CPV/>

SELECT ?genere (COUNT(DISTINCT ?via) AS ?numero_strade)
WHERE {
  ?via a clv:Street ;
       clv:isDedicatedTo ?persona .
  ?persona a cpv:Person ;
           cpv:sex ?genere .
}
GROUP BY ?genere
ORDER BY DESC(?numero_strade)

Risultati:

?genere?numero_strade
Male1066
Female66

Le 828 strade classificate come Toponimo non hanno un nodo cpv:Person associato e non compaiono in questa query, che filtra solo i nodi con cpv:sex.

Query 3 — Tipologia delle strade femminili

Domanda: le strade intitolate a donne sono prevalentemente "Via" o "Piazza/Piazzale"? La tipologia riflette una gerarchia simbolica nel riconoscimento pubblico.
Keyword: UNION, FILTER, DISTINCT, ORDER BY

"su">"sc">## Query 3 — Strade dedicate a donne per tipologia (Via vs Piazza/Piazzale)"su">
"su">"sc">## Keywords: UNION, FILTER, DISTINCT, ORDER BY"su">
"su">"sc">##"su">
"su">"sc">## Domanda: le strade intitolate a donne sono prevalentemente"su">
"su">"sc">## "Via", "Piazza" o "Piazzale"? La tipologia di intitolazione"su">
"su">"sc">## (strada minore vs spazio pubblico) riflette un'ulteriore"su">
"su">"sc">## gerarchia simbolica nel riconoscimento delle donne."su">

PREFIX clv: <https://w3id.org/italia/onto/CLV/>
PREFIX cpv: <https://w3id.org/italia/onto/CPV/>

SELECT DISTINCT ?tipologia ?nomeVia ?nomePersona
WHERE {
  {
    ?via a clv:Street ;
         clv:hasStreetName ?nomeVia ;
         clv:isDedicatedTo ?persona .
    ?persona cpv:fullName ?nomePersona ;
             cpv:sex "Female" .
    FILTER(STRSTARTS(?nomeVia, "VIA "))
    BIND("Via" AS ?tipologia)
  }
  UNION
  {
    ?via a clv:Street ;
         clv:hasStreetName ?nomeVia ;
         clv:isDedicatedTo ?persona .
    ?persona cpv:fullName ?nomePersona ;
             cpv:sex "Female" .
    FILTER(STRSTARTS(?nomeVia, "PIAZZA ") || STRSTARTS(?nomeVia, "PIAZZALE "))
    BIND("Piazza / Piazzale" AS ?tipologia)
  }
}
ORDER BY ?tipologia ?nomePersona

Risultati aggregati:

?tipologiaConteggio
Via45
Piazza / Piazzale8

Le restanti intitolazioni femminili sono Largo, Passaggio, Rotonda, Vicolo. Solo 8 donne hanno una piazza o un piazzale a loro nome.

Query 5 — OPTIONAL: persone con più strade dedicate

Domanda: ci sono personaggi a cui sono dedicate più strade? OPTIONAL mostra anche le persone con una sola strada (con ?nomeVia2 non valorizzato).
Keyword: OPTIONAL, DISTINCT, FILTER, ORDER BY, LIMIT

"su">"sc">## Query 5 — Personaggi con nome via e dati biografici opzionali"su">
"su">"sc">## Keywords: OPTIONAL, DISTINCT, FILTER, ORDER BY"su">
"su">"sc">##"su">
"su">"sc">## Domanda: quali persone sono onorate con più di una strada?"su">
"su">"sc">## OPTIONAL viene usato per cercare una seconda intitolazione:"su">
"su">"sc">## se esiste, viene mostrata; se non esiste, la riga appare ugualmente"su">
"su">"sc">## (con ?nomeVia2 = non valorizzato)."su">
"su">"sc">##"su">
"su">"sc">## Questo pattern OPTIONAL è utile anche per query future quando"su">
"su">"sc">## il KG verrà esteso con dati biografici (nascita, professione)"su">
"su">"sc">## che non saranno disponibili per tutti i personaggi."su">

PREFIX clv: <https://w3id.org/italia/onto/CLV/>
PREFIX cpv: <https://w3id.org/italia/onto/CPV/>

SELECT DISTINCT ?nomePersona ?genere ?nomeVia1 ?nomeVia2
WHERE {
  ?via1 a clv:Street ;
        clv:hasStreetName ?nomeVia1 ;
        clv:isDedicatedTo ?persona .
  ?persona a cpv:Person ;
           cpv:fullName ?nomePersona ;
           cpv:sex ?genere .

  OPTIONAL {
    ?via2 a clv:Street ;
          clv:hasStreetName ?nomeVia2 ;
          clv:isDedicatedTo ?persona .
    FILTER(?via2 != ?via1)
  }

  FILTER(?genere = "Female" || ?genere = "Male")
}
ORDER BY ?genere ?nomePersona
LIMIT 200

Query 6 — Dati biografici di tutte le persone onorate

Domanda: quali sono la professione, la data di nascita e la data di morte delle persone a cui Bologna ha dedicato una strada? I dati provengono dall'arricchimento biografico via Wikidata (copertura 88,7% su 1.129 persone).
Keyword: OPTIONAL, SELECT DISTINCT, FILTER, BOUND, ORDER BY, LIMIT

"su">"sc">## Query 6 — Dati biografici: professione, data di nascita e morte per tutte le persone"su">
"su">"sc">## Mostra le proprietà ex: aggiunte tramite arricchimento Wikidata (88,7% copertura)."su">
"su">"sc">## OPTIONAL perché non tutte le persone hanno dati completi."su">
"su">"sc">## Keyword: OPTIONAL, SELECT DISTINCT, WHERE, FILTER, ORDER BY, LIMIT"su">
PREFIX clv:  <https://w3id.org/italia/onto/CLV/>
PREFIX cpv:  <https://w3id.org/italia/onto/CPV/>
PREFIX ex:   <https://w3id.org/bologna/ontology#>

SELECT DISTINCT ?nomeVia ?nomePersona ?genere ?professione ?dataNascita ?dataMorte ?luogoNascita
WHERE {
  ?strada  a clv:Street ;
           clv:hasStreetName ?nomeVia ;
           clv:isDedicatedTo ?persona .
  ?persona cpv:fullName ?nomePersona ;
           cpv:sex      ?genere .
  OPTIONAL { ?persona ex:professione   ?professione  }
  OPTIONAL { ?persona ex:dataNascita   ?dataNascita  }
  OPTIONAL { ?persona ex:dataMorte     ?dataMorte    }
  OPTIONAL { ?persona ex:luogoNascita  ?luogoNascita }
  FILTER ( BOUND(?professione) || BOUND(?dataNascita) )
}
ORDER BY ?nomePersona
LIMIT 20
